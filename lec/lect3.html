1. Понятие цикла в алгоритмах

Цикл — это управляющая конструкция, позволяющая многократно выполнять один и тот же блок операторов до тех пор, пока выполняется некоторое условие. Циклы являются основой для обработки массивов, организации ввода-вывода больших объёмов данных, реализации рекуррентных вычислений и многих других задач.

В языке C++ имеются три основных вида циклов:

  • Цикл while — с предусловием.
  • Цикл do-while — с постусловием.
  • Цикл for — параметрический (со счётчиком).

Кроме того, в стандарте C++11 появился диапазонный for (range-based for), упрощающий перебор коллекций.

Важнейшими понятиями, связанными с циклами, являются:

  • Итерация — однократное выполнение тела цикла.
  • Счётчик цикла — переменная, управляющая числом повторений (особенно в циклах for).
  • Условие продолжения — логическое выражение, определяющее, нужно ли выполнять очередную итерацию.
  • Бесконечный цикл — цикл, условие которого всегда истинно (требует принудительного прерывания).

2. Цикл while (с предусловием)

Цикл while проверяет условие до выполнения тела цикла. Если условие истинно — тело выполняется, после чего проверка повторяется. Если условие ложно изначально, тело не выполнится ни разу.

Синтаксис:

while (условие) {
    // тело цикла (один или несколько операторов)
}

Пример 1: вывод чисел от 1 до 5.

int i = 1;
while (i <= 5) {
    std::cout << i << " ";
    ++i;
}
// Результат: 1 2 3 4 5

Пример 2: суммирование вводимых чисел до тех пор, пока не будет введён ноль.

int sum = 0, value = 0;
std::cout << "Введите числа (0 для завершения): ";
std::cin >> value;
while (value != 0) {
    sum += value;
    std::cin >> value;
}
std::cout << "Сумма: " << sum << std::endl;

Важно: в теле цикла необходимо изменять переменные, влияющие на условие, иначе возможен бесконечный цикл.

3. Цикл do-while (с постусловием)

Цикл do-while сначала выполняет тело цикла, а затем проверяет условие. Если условие истинно, цикл повторяется. Таким образом, тело цикла выполняется как минимум один раз, даже если условие ложно.

Синтаксис:

do {
    // тело цикла
} while (условие);   // обратите внимание на точку с запятой

Пример: меню программы (действие выполняется хотя бы раз).

char choice;
do {
    std::cout << "1. Начать игру\n";
    std::cout << "2. Загрузить\n";
    std::cout << "3. Выйти\n";
    std::cout << "Ваш выбор: ";
    std::cin >> choice;
    // обработка выбора...
} while (choice != '3');

Сравнение while и do-while: do-while гарантирует хотя бы одно выполнение, что удобно для диалоговых программ и ввода данных с проверкой.

4. Цикл for (параметрический цикл)

Цикл for используется, когда заранее известно или может быть вычислено количество итераций. Он объединяет инициализацию счётчика, условие продолжения и изменение счётчика в одной строке.

Синтаксис:

for (инициализация; условие; выражение_шага) {
    // тело цикла
}
  • инициализация — выполняется один раз в начале цикла (обычно объявление и установка счётчика).
  • условие — проверяется перед каждой итерацией; если true — тело выполняется.
  • выражение_шага — выполняется после каждой итерации (обычно инкремент или декремент).

Пример 1: вывод чисел от 1 до 10.

for (int i = 1; i <= 10; ++i) {
    std::cout << i << " ";
}

Пример 2: сумма элементов массива.

int arr[] = {5, 8, 12, 3, 7};
int sum = 0;
for (int i = 0; i < 5; ++i) {
    sum += arr[i];
}
std::cout << "Сумма = " << sum;

Гибкость for: любая из трёх частей может быть опущена. Например, бесконечный цикл:

for (;;) {
    // бесконечный цикл
}

Также можно использовать несколько переменных, разделяя их запятыми:

for (int i = 0, j = 10; i < j; ++i, --j) {
    std::cout << i << " " << j << std::endl;
}

5. Диапазонный for (range-based for, C++11 и новее)

Для перебора всех элементов контейнера (массива, вектора, строки и т.д.) в C++11 был добавлен упрощённый цикл for. Он автоматически перебирает каждый элемент без необходимости использовать индексы.

Синтаксис:

for (тип переменная : контейнер) {
    // тело цикла
}

Пример: вывод массива.

int numbers[] = {10, 20, 30, 40};
for (int x : numbers) {
    std::cout << x << " ";
}
// Вывод: 10 20 30 40

Для изменения элементов контейнера используйте ссылку:

for (int &x : numbers) {
    x *= 2;   // удваиваем каждый элемент
}

Диапазонный for особенно удобен при работе со стандартными библиотечными контейнерами (std::vector, std::array, std::string).

6. Управление циклом: break и continue

Внутри любого цикла можно использовать два специальных оператора для изменения естественного хода выполнения:

  • break — немедленно завершает цикл (выход из самого внутреннего цикла).
  • continue — пропускает оставшуюся часть текущей итерации и переходит к следующей итерации (проверке условия для while/do-while или к шагу в for).

Пример с break: поиск первого отрицательного числа.

int data[] = {5, 2, -1, 8, 4};
int index = -1;
for (int i = 0; i < 5; ++i) {
    if (data[i] < 0) {
        index = i;
        break;   // выход из цикла
    }
}
if (index != -1) {
    std::cout << "Первое отрицательное число на позиции " << index;
}

Пример с continue: вывод только чётных чисел от 1 до 10.

for (int n = 1; n <= 10; ++n) {
    if (n % 2 != 0) {
        continue;   // пропускаем нечётные
    }
    std::cout << n << " ";
}
// Вывод: 2 4 6 8 10

Важно: чрезмерное использование break и continue может ухудшить читаемость кода. Старайтесь проектировать условия цикла так, чтобы они были естественными.

7. Вложенные циклы

Циклы могут содержать внутри себя другие циклы. Это широко используется при работе с многомерными массивами, генерации таблиц, реализации алгоритмов сортировки (например, пузырьковая сортировка).

Пример: таблица умножения 5×5.

for (int i = 1; i <= 5; ++i) {
    for (int j = 1; j <= 5; ++j) {
        std::cout << i * j << "\t";
    }
    std::cout << std::endl;
}

Вложенные циклы требуют внимания к сложности алгоритма: если внешний цикл выполняется N раз, а внутренний M раз, то общее количество итераций — N * M.

Пример: поиск пар элементов, сумма которых равна заданному числу.

int arr[] = {1, 3, 5, 7, 9};
int target = 10;
for (int i = 0; i < 5; ++i) {
    for (int j = i + 1; j < 5; ++j) {
        if (arr[i] + arr[j] == target) {
            std::cout << "Пара: (" << arr[i] << ", " << arr[j] << ")" << std::endl;
        }
    }
}

8. Бесконечные циклы и способы их предотвращения

Бесконечный цикл возникает, если условие продолжения цикла никогда не становится ложным. Иногда бесконечные циклы используются намеренно (например, в серверных приложениях, в играх до выхода), но чаще это ошибка.

Типичные ошибки:

  • Забыт инкремент счётчика: while (i < 10) { ... } без i++.
  • Неправильное условие: for (int i = 0; i >= 0; ++i).
  • Сравнение чисел с плавающей точкой на точное равенство: while (x != 1.0) может никогда не стать ложным из-за погрешностей.

Пример корректного бесконечного цикла с выходом по break:

int password;
while (true) {
    std::cout << "Введите пароль: ";
    std::cin >> password;
    if (password == 1234) {
        std::cout << "Доступ разрешён." << std::endl;
        break;
    } else {
        std::cout << "Неверно, попробуйте снова." << std::endl;
    }
}

Такой паттерн часто используется для ввода с проверкой, когда точное количество попыток неизвестно.

9. Сравнение и выбор типа цикла

Тип цикла Когда использовать Особенности
while Количество итераций неизвестно заранее, зависит от условия, может быть ноль итераций. Проверка перед выполнением.
do-while Тело должно быть выполнено хотя бы один раз (например, меню, ввод с проверкой). Проверка после выполнения.
for Чётко известен диапазон или количество повторений (работа со счётчиком, массивами). Компактная запись инициализации, условия и шага.
диапазонный for Перебор всех элементов контейнера без индексов. Удобен и безопасен, но не даёт доступа к индексу элемента.

Выбор правильного типа цикла делает программу более понятной и менее подверженной ошибкам.

11. Распространённые ошибки при работе с циклами

  • Ошибка на единицу (off-by-one): неправильное граничное условие (например, i <= n вместо i < n при обходе массива).
  • Использование неинициализированных переменных в условии цикла.
  • Изменение переменной цикла внутри тела непредусмотренным образом (кроме специальных алгоритмов).
  • Точка с запятой после for(...) или while(...) — приводит к пустому телу цикла.
  • Неучёт типа данных при больших значениях счётчика (переполнение).
  • Побочные эффекты в условии цикла (например, вызов функции, изменяющей состояние).

Всегда тестируйте циклы на граничных значениях: 0, 1, максимальное значение.
