Здесь есть все, что ты хочешь

1. Понятие функции в алгоритмизации

В любом алгоритме часто встречаются повторяющиеся действия. Функция — это логически обособленный блок кода, имеющий имя, который можно многократно вызывать из разных частей программы. Функции позволяют:

  • избежать дублирования кода (DRY-принцип: Don't Repeat Yourself);
  • разбить сложную задачу на мелкие подзадачи (модульность);
  • упростить тестирование и отладку;
  • повысить читаемость программы.

В C++ каждая программа обязательно содержит функцию main() — точку входа. Все остальные функции создаются программистом или берутся из стандартных библиотек.

2. Определение и вызов функции. Синтаксис

Общая форма определения функции:

тип_возвращаемого_значения имя_функции(список_параметров) {
    // тело функции
    return значение;   // если тип_возврата не void
}
  • тип_возвращаемого_значения — тип данных, который функция возвращает (int, double, bool, void и т.д.). Если функция не возвращает значение, указывается void.
  • имя_функции — идентификатор, по которому вызывается функция.
  • список_параметров — перечень переменных (с типами), принимающих входные данные. Может быть пустым (() или (void)).
  • тело функции — блок операторов в фигурных скобках.
  • return — оператор, завершающий функцию и возвращающий значение (если тип не void).

Пример простейшей функции:

// Функция сложения двух целых чисел
int sum(int a, int b) {
    return a + b;
}

Вызов функции:

#include <iostream>

int sum(int a, int b);   // объявление (прототип)

int main() {
    int result = sum(5, 3);   // вызов функции
    std::cout << "Сумма = " << result << std::endl;
    return 0;
}

int sum(int a, int b) {       // определение
    return a + b;
}

Прототип (объявление) функции сообщает компилятору о её существовании до фактического определения. Это позволяет вызывать функцию до её реализации.

3. Параметры и аргументы. Способы передачи

Параметры — это переменные в заголовке функции. Аргументы — конкретные значения, передаваемые при вызове.

В C++ существуют три основных способа передачи параметров:

3.1. Передача по значению (по умолчанию)

Функция получает копию аргумента. Изменения параметра внутри функции не влияют на исходную переменную.

void changeValue(int x) {
    x = 100;   // меняется копия
}

int main() {
    int a = 5;
    changeValue(a);
    std::cout << a;   // выведет 5, а не 100
}

3.2. Передача по ссылке (&)

Параметр становится псевдонимом переданной переменной. Изменения внутри функции отражаются на оригинале.

void swap(int &x, int &y) {
    int temp = x;
    x = y;
    y = temp;
}

int main() {
    int a = 3, b = 7;
    swap(a, b);
    std::cout << a << " " << b;   // 7 3
}

3.3. Передача по указателю (*)

Функция получает адрес переменной. Через разыменование можно изменить оригинал. В современных программах предпочтительнее ссылки.

void setToZero(int *p) {
    if (p != nullptr) *p = 0;
}

int main() {
    int value = 42;
    setToZero(&value);
    std::cout << value;   // 0
}

Рекомендация: Для входных параметров, которые не нужно изменять, используйте передачу по значению или константную ссылку (const T&). Для изменения — ссылку.

4. Возврат значений. Оператор return

Оператор return завершает выполнение функции и возвращает значение вызывающему коду. Тип возвращаемого значения должен соответствовать объявленному типу функции.

Примеры:

int max(int a, int b) {
    if (a > b)
        return a;
    else
        return b;
}

// Функция может иметь несколько return, но только один из них сработает.

Для функций void оператор return; не обязателен, но может использоваться для досрочного выхода.

void printPositive(int n) {
    if (n <= 0)
        return;   // досрочный выход
    std::cout << n << std::endl;
}

Важно: Нельзя вернуть ссылку на локальную переменную (она разрушится после выхода из функции).

5. Аргументы по умолчанию

C++ позволяет задавать параметрам значения по умолчанию. Если при вызове аргумент не указан, используется значение по умолчанию.

Правила: параметры со значениями по умолчанию должны располагаться в конце списка параметров.

void greet(std::string name, std::string prefix = "Здравствуйте") {
    std::cout << prefix << ", " << name << "!" << std::endl;
}

int main() {
    greet("Анна");                 // "Здравствуйте, Анна!"
    greet("Иван", "Привет");      // "Привет, Иван!"
}

6. Перегрузка функций (Function Overloading)

В C++ можно определять несколько функций с одним именем, но разными параметрами (количество, типы или порядок). Компилятор сам выбирает нужную версию на основе переданных аргументов.

Пример перегрузки:

#include <iostream>

int multiply(int a, int b) {
    return a * b;
}

double multiply(double a, double b) {
    return a * b;
}

int multiply(int a, int b, int c) {
    return a * b * c;
}

int main() {
    std::cout << multiply(2, 3) << std::endl;        // 6 (int, int)
    std::cout << multiply(2.5, 3.1) << std::endl;  // 7.75 (double, double)
    std::cout << multiply(2, 3, 4) << std::endl;    // 24 (три параметра)
}

Перегрузка не может отличаться только типом возвращаемого значения.

7. Рекурсивные функции

Рекурсия — это вызов функцией самой себя. Рекурсивные функции позволяют элегантно решать задачи, которые сводятся к аналогичным подзадачам (вычисление факториала, обход деревьев, алгоритм Евклида и др.).

Обязательные компоненты рекурсии:

  • Базовый случай (условие выхода).
  • Рекурсивный шаг (вызов функции с изменёнными параметрами).

Пример: вычисление факториала.

int factorial(int n) {
    if (n <= 1)       // базовый случай
        return 1;
    else
        return n * factorial(n - 1);   // рекурсивный вызов
}

Каждый рекурсивный вызов создаёт новый стековый фрейм. Глубокая рекурсия может привести к переполнению стека. В таких случаях используют итеративные алгоритмы.

Пример: числа Фибоначчи (рекурсивная версия неэффективна).

int fib(int n) {
    if (n <= 1) return n;
    return fib(n - 1) + fib(n - 2);
}

Для улучшения производительности используют мемоизацию или итеративный подход.

8. Inline-функции

inline — это подсказка компилятору: попытаться вставить код функции непосредственно в место вызова, вместо создания вызова. Это может увеличить производительность для коротких функций, но увеличивает размер кода.

inline int square(int x) {
    return x * x;
}

Современные компиляторы часто игнорируют inline и сами решают, что оптимизировать. Используйте inline для очень маленьких функций в заголовочных файлах.

9. Лямбда-функции (анонимные функции) — краткое знакомство

Начиная с C++11, появились лямбда-выражения — локальные функции, которые можно определить прямо в месте использования. Они удобны для передачи в алгоритмы стандартной библиотеки.

Минимальный синтаксис:

[capture](parameters) -> return_type { body }

Пример:

#include <iostream>
#include <vector>
#include <algorithm>

int main() {
    std::vector<int> v = {1, 2, 3, 4, 5};
    // лямбда, возвращающая true для четных чисел
    auto isEven = [](int n) { return n % 2 == 0; };
    int count = std::count_if(v.begin(), v.end(), isEven);
    std::cout << "Четных чисел: " << count << std::endl;
}

Лямбды подробно изучаются после освоения основ классов и алгоритмов STL.

10. Область видимости переменных. Локальные и глобальные переменные

Переменные, объявленные внутри функции, называются локальными. Они создаются при входе в функцию и уничтожаются при выходе. Глобальные переменные объявляются вне всех функций и видны во всей программе (что часто считается плохим стилем).

#include <iostream>

int globalVar = 100;   // глобальная переменная

void func() {
    int localVar = 10; // локальная переменная
    std::cout << "globalVar = " << globalVar << std::endl;
    // localVar недоступна за пределами func
}

Статические локальные переменные (static) сохраняют своё значение между вызовами функции.

void counter() {
    static int calls = 0;
    calls++;
    std::cout << "Функция вызвана " << calls << " раз" << std::endl;
}

11. Функции и массивы. Передача массивов в функцию

При передаче массива в функцию он «вырождается» в указатель на первый элемент. Размер массива нужно передавать отдельно.

void printArray(int arr[], int size) {
    for (int i = 0; i < size; ++i) {
        std::cout << arr[i] << " ";
    }
}

int main() {
    int data[] = {10, 20, 30, 40};
    int size = sizeof(data) / sizeof(data[0]);
    printArray(data, size);
}

Для многомерных массивов необходимо указывать размерность, кроме первой.

12. Стандартные математические функции (библиотека <cmath>)

Ранее мы уже касались математических функций. Они также являются примерами готовых функций:

#include <cmath>
double x = 4.0;
double root = sqrt(x);   // 2.0
double power = pow(2, 3); // 8.0

Все они следуют тем же правилам: передача параметров, возврат значения.

13. Прототипы функций и раздельная компиляция

В реальных проектах функции часто объявляют в заголовочных файлах (.h или .hpp), а определяют в файлах реализации (.cpp). Это позволяет компилировать части программы отдельно.

Пример структуры:

  • math_utils.h — содержит прототипы функций.
  • math_utils.cpp — определения функций.
  • main.cpp — использует функции, подключая заголовок.
// math_utils.h
#ifndef MATH_UTILS_H
#define MATH_UTILS_H
int add(int a, int b);
#endif

// math_utils.cpp
#include "math_utils.h"
int add(int a, int b) { return a + b; }

// main.cpp
#include "math_utils.h"
#include <iostream>
int main() { std::cout << add(2,3); }

14. Типичные ошибки при работе с функциями

  • Несоответствие типа возврата: объявлено int, а возвращается double — происходит усечение или ошибка.
  • Забыт оператор return в не-void функции — неопределённое поведение.
  • Возврат ссылки на локальную переменную — висячая ссылка.
  • Рекурсия без базового случая — бесконечная рекурсия, переполнение стека.
  • Передача массива без указания размера — потеря информации о длине.
  • Конфликт имён при перегрузке — компилятор не может выбрать подходящую версию.

15. Практический пример: библиотека геометрических функций

Создадим несколько полезных функций для работы с геометрическими фигурами.

#include <iostream>
#include <cmath>

const double PI = 3.141592653589793;

// Площадь круга
double circleArea(double radius) {
    return PI * radius * radius;
}

// Длина окружности
double circleCircumference(double radius) {
    return 2 * PI * radius;
}

// Расстояние между двумя точками на плоскости
double distance(double x1, double y1, double x2, double y2) {
    return std::sqrt(std::pow(x2 - x1, 2) + std::pow(y2 - y1, 2));
}

int main() {
    double r = 5.0;
    std::cout << "Площадь круга: " << circleArea(r) << std::endl;
    std::cout << "Расстояние между (0,0) и (3,4): " << distance(0,0,3,4) << std::endl;
    return 0;
}

Этот пример показывает, как функции инкапсулируют логику и могут быть переиспользованы.
